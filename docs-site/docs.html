Documentation

Portly is a developer CLI that exposes localhost to the public internet using a tunnel provider. This reference covers command syntax, internal flow, behavior guarantees, and deployment recommendations.

CLI Commands

CommandDescriptionResult
npx @solez-ai/portlyAuto-detect a running local server from common ports.Starts tunnel and prints public URL.
npx @solez-ai/portly 3000Use manual port selection.Bypasses scan, tunnels explicit port.
npx @solez-ai/portly 3000 --name saminUse manual port and custom subdomain.Requests named tunnel from provider.
npx @solez-ai/portly --host yourdomain.comCustomize display host branding.Use only with wildcard DNS/routing support.

Port Detection Strategy

When no port is provided, Portly probes this default list in order:

3000, 5173, 8080, 8000, 4200, 4000, 5000

Each port receives an HTTP probe against http://127.0.0.1:<port>. If a response is returned, Portly marks that port active and attempts a lightweight framework hint by checking headers/body for known signatures.

Session Management

Manual Termination

CTRL+C cleanly shuts down the active tunnel and exits the process.

Max Duration

Default session timeout is 2 hours from tunnel start.

Inactivity Window

Inactivity monitoring closes the tunnel after 2 hours with no request events.

Internal Flow

1. User runs command
2. CLI resolves port (manual or auto-detect)
3. CLI requests tunnel from LocalTunnel provider
4. Provider returns live public endpoint
5. CLI prints branded URL + QR code
6. Session remains active until closed or timed out

Domain and Cloudflare Setup

1. Use your free Pages domain

Your live docs are hosted at https://portly-live.pages.dev.

2. Deploy docs site on Cloudflare Pages

Use docs-site as the build output directory in Cloudflare Pages settings.

3. Wildcard links require custom tunnel infra

Only configure wildcard DNS (for example *.yourdomain.com) when you run custom tunnel routing.

4. Enforce HTTPS

Enable Full (strict) TLS mode in Cloudflare and verify certificate status in the dashboard.

HTTPS / Not Secure Troubleshooting

If you see Not Secure, verify that you are on https://portly-live.pages.dev exactly.

Cloudflare Pages does not automatically provide wildcard SSL for *.portly-live.pages.dev.

For tunnel links, use the provider URL (for example https://name.loca.lt) unless you run custom wildcard infrastructure.
